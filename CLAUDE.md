# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

The course repository for Ed Donner's "Master AI Agentic Engineering" (refreshed Summer 2026) — a 6-week, hands-on course on building autonomous AI agents. Each numbered top-level directory is one week and introduces a different agent framework, building from raw API calls up to multi-agent systems with MCP. It is teaching material, not a single deployable application: most "code" lives in Jupyter notebook labs (`N_labX.ipynb`) plus standalone scripts and apps that the labs build toward.

- `1_foundations/` — raw OpenAI SDK calls, prompt patterns, the personal "twin" Gradio chatbot.
- `2_openai/` — OpenAI Agents SDK; the `deep_research/` multi-agent research app.
- `3_crewai/` — CrewAI; see its own README and the `reference/` projects.
- `4_langchain_langgraph/` — LangChain + LangGraph; the `sidekick` agent and `app.py`.
- `5_agent_frameworks/` — Google ADK + A2A, Strands, Pydantic AI, Microsoft Agent Framework, Agno, Mastra (TypeScript), and a from-scratch `5_agent_loop/`.
- `6_mcp/` — MCP; an autonomous trading-floor simulation (Python backend + Vite/TS frontend).
- `community_contributions/` folders (at root and inside most weeks) — student-submitted variations; treat as untrusted/unreviewed and keep changes out unless explicitly asked.

## Environment & running code

This repo is managed with **uv** (Python 3.12). Run everything through uv from the appropriate directory:

```bash
uv sync                              # install all root dependencies
uv run jupyter lab                   # open the notebook labs
uv run python <path/to/script.py>    # run a standalone script/app
```

- Single root `pyproject.toml` + `uv.lock` cover every week's dependencies — there is no per-week venv except where a subproject ships its own `pyproject.toml`/`requirements.txt` (e.g. the CrewAI `reference/` crews, `1_foundations/twin/`, `2_openai/deep_research/`).
- **Secrets** live in a `.env` file at the repo root (gitignored; `.cursorignore` also excludes it). Code loads it with `python-dotenv`. The course uses many provider keys (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `DEEPSEEK_API_KEY`, `GOOGLE_API_KEY`, etc.) — any given lab only needs the ones it calls. Models are swappable to cheaper/free alternatives (DeepSeek, OpenRouter, Ollama) via base-URL overrides; see `guides/09_ai_apis_and_ollama.ipynb`.
- Setup instructions per OS are in `setup/SETUP-{mac,pc,linux}.md`; cross-cutting how-tos are in `guides/`.

There is no repo-wide test/lint/build step — this is course material, not a packaged product. Validate work by running the relevant notebook or app.

Several weeks build a Gradio web UI launched by running its module directly:

```bash
uv run python 1_foundations/twin/app.py        # personal "digital twin" chatbot

```

## CrewAI (week 3) — read first

CrewAI is installed as a **uv tool**, pinned to match the course: `uv tool install crewai==1.14.4`. The instructor's projects are under `3_crewai/reference/`; students build their own under `3_crewai/coursework/`. See `3_crewai/README.md` for the full workflow.

- Scaffold a new crew from the `3_crewai` directory: `crewai create crew <name>`.
- Run a crew from inside its project directory: `crewai run`.
- Each crew is a standard CrewAI layout: agents/tasks defined in `src/<name>/config/{agents,tasks}.yaml`, wiring in `crew.py`, entrypoint in `main.py`, custom tools in `tools/`.
- The `coder` and `engineering_team` crews execute generated code inside **Docker** — Docker must be running before `crewai run`. The other crews don't need it.
- Each crew ships an `AGENTS.md` (auto-generated by `crewai create`) that mandates checking the installed CrewAI version and live docs before writing CrewAI code, because the API moves fast and training data is often stale. Follow it.

## Other framework conventions

- **Week 5** subprojects each include a `SWAP_AI.md` explaining how to repoint that day's framework at a different model/provider (e.g. ADK via LiteLLM with an `openai/` prefix). The day's tools and logic are model-agnostic — typically only one model line changes. `5_agent_frameworks/4_mastra/` is a TypeScript/Node project (`npm install`, scripts in `package.json`).
- **Week 6 trading floor** (`6_mcp/`): the backend (`backend/`) is a set of MCP servers (`*_server.py`), account/market simulation, and `trading_floor.py` — a scheduler that runs autonomous traders on an interval (`uv run python 6_mcp/backend/trading_floor.py`, runs forever). The frontend (`frontend/`) is a Vite + TypeScript app (`npm run dev`).

## Working in this repo

- Match the surrounding lab's style — these files are read by learners, so keep examples clear and self-contained.
- Keep edits scoped to the week/subproject in question; the weeks are independent and share only the root dependency set.
- When asked to add a community contribution, keep it concise and place it under the relevant `community_contributions/` directory.
